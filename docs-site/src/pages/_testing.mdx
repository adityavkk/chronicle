<section class="block" id="philosophy">
  <SectionHead num="01" label="Philosophy" title="A pure core behind an <em>imperative shell</em>" />

  Chronicle splits every component into a pure deterministic core and an
  imperative I/O shell, then tests each at the cheapest level that is still
  faithful.[^split] The core is a function over values — `ValidateProducer`, the
  offset and frame codecs, the webhook `FenceDecision`/`ClaimDecision`/`MergeAcks`
  mirrors, the porcupine fence model, the cursor checker. It has no Redis, no
  clock, no cluster, so it is unit-tested with table-driven hand-built inputs and
  runs in milliseconds. The shell is where the bytes actually move — the Lua
  scripts, the pub/sub wakeups, the durable hashes and lease ZSETs — and it is
  covered by integration tests against a real Redis on an isolated database.

  Above the single process, properties that only emerge across processes — the
  single-holder invariant, cursor monotonicity, end-to-end delivery under crashes
  — get fault injection on a live `k3d` cluster, with `porcupine` model-checking
  the recorded histories. The HTTP surface runs against an in-memory store via
  `httptest`. And the protocol is pinned from outside the tree by a vendored
  black-box conformance suite.

  <Table>
    | Layer | What runs | Faults | Infra |
    | --- | --- | --- | --- |
    | **Pure core** | unit tables, porcupine model, cursor checker | none — deterministic | none |
    | **I/O shell** | store / webhook integration, the Lua | timing, durability across a fresh store | real Redis (db 15 / 14) |
    | **HTTP surface** | `handler_test.go` via `httptest` | concurrency via goroutine + sleep | in-memory store |
    | **Cross-process** | jepsen scenarios, porcupine histories | pod kill, Redis kill, gcPause | live `k3d` cluster |
    | **Wire contract** | conformance (vitest + fast-check) | property fuzzing | black-box vs a live server |
  </Table>

  The shape of the pyramid is deliberate. The bulk of the assertions live in the
  cheap, deterministic cores; the expensive cluster tests carry only the
  properties that genuinely require concurrency and crashes. A single rule is
  often pinned more than once — `ValidateProducer` from both backends, the fence
  from Go and Lua and the checker — so a regression has to defeat two independent
  assertions to slip through.

  <Callout icon="info" label="Why the split pays off here">
    The most safety-critical logic in Chronicle — the producer fence, the
    subscription generation fence, the forward-only cursor — is exactly the logic
    that is hardest to test against a live system, because the interesting cases
    are rare races. Lifting that logic into a pure core lets the rare case become
    a hand-built table row: a stale bearer token where <code>reqGen</code> matches
    but <code>tokenGen</code> is stale, a post-expiry <code>FENCED</code> that
    must still linearize. The live drivers cannot reliably reach those states; the
    unit tests reach them by construction.
  </Callout>
</section>

<section class="block" id="store">
  <SectionHead num="02" label="Store" title="The durable log, <em>two backends</em>" />

  The store layer is the append-only durable streams log behind one `Store`
  interface, with two backends: an in-process `MemoryStore` in pure Go, and a
  Redis-backed store built on ZSET frames, hash-tagged keys, and Lua. The pure
  half (`store/*`, the offset and producer codecs) needs no Redis; the Redis half
  (`store/redis/*` integration, fork, notify, differential) needs a real server,
  gated by `-short` and `FlushDB`-once.[^realredis] The keys and metadata tests
  are pure encoding and run anywhere.

  <Table wide>
    | Test group | Verifies | Code ref |
    | --- | --- | --- |
    | Store contract: create / idempotency / append-read / delete | `Create` is idempotent on matching config and `ErrConfigMismatch` otherwise; append returns a monotonic offset; read-from-tail is empty + up-to-date; delete+recreate is fresh | [^contract] |
    | Opaque lexicographic offset | two zero-padded 16-digit fields joined by `_`, so bytewise string order equals semantic order; parse rejects URL-hostile chars; round-trips | [^offset] |
    | Producer fencing state machine (pure oracle) | PROTOCOL §5.2.1: first contact is seq 0; stale epoch is `ErrStaleEpoch`; an epoch bump must restart at seq 0; duplicate returns the highest accepted seq | [^producer] |
    | **DIFFERENTIAL** parity: Redis Lua vs pure-Go oracle | the Lua append path is byte-identical to `ValidateProducer` per row, plus the persisted hash matches and a write lands iff the row is accepted | [^differential] |
    | Forks / branching | stitched reads, JSON vs binary sub-offsets, `RefCount`, soft-delete-through, and a hard cascade that wipes every Redis key | [^fork] |
    | TTL & expiry | sliding window renewed by Read/Append but not Get; absolute `ExpiresAt`; fork TTL inheritance; expired-source-with-forks flips to soft-deleted | [^ttl] |
    | Redis keyspace & hash-tag co-location | the `meta`/`msg`/`prod`/`forks` keys per stream share one hash tag for one cluster slot; user braces escaped injectively | [^keys] |
    | Frame encoding & lex bounds | an `offset` then `|` then raw bytes, stored in a ZSET at score 0; round-trips adversarial bytes; sorting frames equals sorting by offset | [^frame] |
    | Notify / long-poll & SSE wakeups | `WaitForMessages` wakes in under a second via pub/sub, returns immediately on present data, times out at tail, wakes on close, honors cancellation | [^notify] |
    | Stream closure & seq dedup | `CloseStream` idempotent with a stable `FinalOffset`; append-to-closed is `ErrStreamClosed`; seq dedup is bytewise lex (`"10"` after `"2"` is rejected) | [^closure] |
    | Redis metadata round-trip | `StreamMetadata` ↔ hash fields losslessly, nanosecond `ExpiresAt` fidelity; minimal meta leaves optional fields absent | [^meta] |
  </Table>

  The foundation under the whole ZSET model is one property: bytewise frame order
  must equal stream offset order, whatever the payload bytes are. The frame and
  offset tests pin it with a sort over adversarial payloads — `0x00`, `0xff`, and
  the `|` separator itself — rather than trusting well-behaved input.

  ```go title="store/redis/keys_test.go — bytewise order is stream order"
  // a shuffled frame list, payloads chosen to be hostile to string sorting
  frames := []string{
      encodeFrame(off(5),   bytes.Repeat([]byte{0xff}, 10)),
      encodeFrame(off(100), []byte{0x00}),
      encodeFrame(off(7),   []byte("|||")),
      encodeFrame(off(50),  []byte{}),
  }
  sort.Strings(frames)                 // pure lexicographic sort
  got := decodeOffsets(frames)
  // => 5, 7, 50, 100  — offset order, independent of payload bytes
  ```

  The single most load-bearing store test is the differential one. The pure-Go
  `ValidateProducer` table is the oracle; the Redis Lua append path is asserted to
  produce a byte-identical decision for every row of the §5.2.1 table — and not
  only the return value, but the persisted producer hash and the fact that a write
  lands on the data plane *iff* the row is accepted.

  ```go title="store/redis/differential_test.go — Lua subject vs Go oracle, per row"
  for _, row := range producerTable {            // the same 10 rows as producer_test
      wantRes, wantState, wantErr := store.ValidateProducer(row.state, row.epoch, row.seq, now)
      hset(prodKey(p), "p", encode(row.state))   // seed the Redis producer state
      gotRes, gotErr := s.Append(p, payload, prodOpts(row))  // the Lua subject
      assert(gotRes == wantRes && errors.Is(gotErr, wantErr))
      assert(decodeProducer(hget(prodKey(p))) == wantState)  // persisted state matches
      assert(getCurrentOffset(p).advanced == (row.result == Accepted)) // write iff accepted
  }
  ```

  <Callout icon="alert" label="What the store layer does not test">
    Despite the repository name, there is no linearizability, Jepsen-style, or
    <code>go test fuzz</code> coverage in the store layer; offset and frame
    coverage is example-based tables, not generative. There is no concurrent
    multi-writer race or <code>-race</code>-specific store test here. And
    memory-vs-Redis parity is <em>structural</em> — both back the same
    <code>Store</code> interface and share the producer table — not a single
    harness; most contract cases are duplicated per-backend, so a divergence
    relies on hand-kept mirroring.
  </Callout>
</section>

<section class="block" id="http">
  <SectionHead num="03" label="HTTP" title="The protocol surface, <em>in memory</em>" />

  The HTTP layer (`handler_test.go`, 57 functions) and the pure protocol helpers
  (`protocol/protocol_test.go`) cover the stream wire surface: PUT create/fork,
  HEAD, POST append/close, idempotent producers, DELETE/soft-delete, GET catch-up
  and conditional reads, JSON-mode flattening, long-poll, SSE framing, CORS and
  security headers, `Mount` prefix translation, and the TTL/sub-offset/cursor
  codecs. Every handler test runs against an in-memory `store.NewMemoryStore`
  through a shared `do()` helper that returns a `*httptest.ResponseRecorder` — no
  real Redis, no miniredis, no mocks in this layer.[^httpharness]

  <Table wide>

  Concurrency is exercised with a goroutine that appends after a sleep while the
  main goroutine blocks. The one place a recorder is not enough is streaming
  *order*: the SSE drain test spins a real `httptest.NewServer` and asserts the
  final append is emitted on the wire before the close control.

  ```go title="handler_test.go — SSE must drain the last append before it closes"
  srv := httptest.NewServer(h)              // a real server, not a recorder
  body := httpGet(srv.URL + "/test?offset=-1&live=sse")
  // a final append + close arrived during the stream:
  assert strings.Index(body, "data:final") < strings.Index(body, "streamClosed")
  // the byte is delivered first; the close control is strictly after it
  ```

  <Callout icon="alert" label="Gaps in the HTTP layer">
    Error envelopes are plain-text <code>http.Error</code> bodies, not a JSON
    problem shape; tests assert status codes and diagnostic headers but never the
    body text, so wording can drift untested. Glob/wildcard matching is not tested
    here (it lives in the webhook package), and <code>__ds</code> subscriptions
    only hit a <code>501</code> stub. There is no fuzz, property, or
    linearizability test in this area — negative-input coverage is hand-enumerated
    table slices, so an untried malformed input may slip through.
  </Callout>
</section>

<section class="block" id="subscriptions">
  <SectionHead num="04" label="Subscriptions" title="The control plane, <em>three tiers</em>" />

  The `__ds` subscription layer turns stream activity into webhook deliveries and
  pull-wakes. Its tests come in three tiers. First, pure unit tests on
  injected-seam functions — the fence/claim/ack mirrors of the Lua, the
  pending-work core, config hashing, glob, retry backoff, and the SSRF URL
  classifier — table-driven and deterministic by passing `now` and the jitter
  fraction explicitly. Second, real-Redis integration tests (db 14, `FlushDB` at
  setup, `t.Skip` under `-short`) that drive the embedded Lua scripts and prove
  durability across a *fresh store over the same Redis*. Third, Manager tests with
  in-memory fakes (`fakeStreams`, `fakeLister`, `fakeMetrics`) over a real store.

  <Table wide>

    | Pending-work / offset-compare core | `tail > acked` over opaque offsets with the `-1`/`""` beginning sentinel; missing streams are not-pending — the same decision the batched sweep and per-link snapshot make | [^sub-pending] |

  The defining technique in this layer is differential parity by convention. Each
  pure mirror in `state.go` carries a doc comment naming the Lua file it must move
  with, and the tests pin both sides to identical decisions. The fence rule —
  rejected unless the token generation, the request generation, and the request
  wake all equal current state — is written once in Lua and mirrored once in Go.

  ```go title="webhook/pure_test.go — the Go mirror of common.lua fenced()"
  // mirror of: fenced() = tok ~= cur or req ~= cur or wake == '' or wake ~= cur
  d := FenceDecision(sub{gen: 7, wake: "w_abc"}, reqGen, reqWake, tokGen)
  // all three equal current      -> ""            (accepted)
  // any stale gen / empty wake    -> ErrCodeFenced (rejected)
  ```

  The crypto tests reconstruct the verifier the way the conformance receiver does
  — re-deriving the `ed25519.PublicKey` from the JWK and verifying over the
  `timestamp.body` string — so the producer and consumer are proven to agree on
  the same wire contract the external suite checks.

  <Callout icon="alert" label="Where the convention can drift">
    Parity here is asserted by convention, not enforced: no test
    <code>EVAL</code>s the Lua and runs the Go mirror on identical inputs to
    assert byte-equal output, so the two sides can silently diverge. True
    concurrency is simulated sequentially — two-worker races are reproduced by
    advancing a clock, not by running in parallel; only the jepsen
    single-holder-linz driver runs genuinely concurrent claims and acks. And the
    SSRF resolve path is only exercised with IP literals, so DNS-rebinding and
    multi-IP hosts are not table-covered.
  </Callout>

<section class="block" id="jepsen">
  <SectionHead num="05" label="Jepsen" title="Killed mid-flight, <em>and linearized</em>" />

  The Jepsen suite (`jepsen/checker/`) is a Go generator + nemesis + checker triad
  over a live `k3d` cluster, and it has two distinct halves. The first is
  final-state durability: kill pods and Redis, let the faults settle, and assert
  every cursor reached its tail — or a deposed worker is `FENCED`. The second is a
  per-history safety suite — porcupine linearizability for the single-holder
  invariant (T1), a non-decreasing checker for cursor monotonicity (T2) — split
  pure-core / imperative-shell, with the pure cores unit-tested on hand-built
  histories and no cluster.[^jepsen-split]

  <Table wide>
    | Test group | Verifies | Code ref |
    | --- | --- | --- |
    | Durability final-state (baseline / origin-restart / redis-restart) | after faults settle, every glob webhook cursor reached its stream tail; duplicates reported (at-least-once) but fence-safe, not failures | [^jep-durability] |
    | Crash-window hardening (arm-crash / glob-create-crash / index-repair) | each hardening slice's property: the sweep re-emits arm-without-emit wakes, the reconcile loop links a stream created under a crash, `ReconcileIndexes` rebuilds a dropped SADD | [^jep-hardening] |
    | Expired-lease-takeover (deterministic) | worker B's takeover rotates the fence to a fresh generation, so A's late ack returns `409 FENCED` — the single-holder property, hand-built | [^jep-takeover] |
    | **T1** single-holder-linz — live porcupine driver | N workers claim/ack under a gcPause nemesis; each outcome is bracketed into a `porcupine.Operation` and `CheckOperationsVerbose` runs the model | [^jep-t1live] |
    | **T1** lease-fence model — the pure oracle | the sequential model of one sub's `(gen, wake, phase)` register; an `OK` ack under a stale token is the two-holder violation | [^jep-t1model] |
    | **T1** model unit tests — legal + violation witnesses | the oracle accepts every legal interleaving and rejects exactly the single-holder violations, from hand-built porcupine histories | [^jep-t1unit] |
    | **T2** cursor-monotonic — live sampling driver | under origin churn, a poller samples each cursor every 150ms; the pure checker reports any backward step after settle | [^jep-t2live] |
    | **T2** cursor-monotonic pure checker + unit tests | per `(sub, path)` the acked offset never decreases; a dip-and-recover reports exactly once; the `-1` sentinel orders below any real offset | [^jep-t2unit] |
  </Table>

  ### The porcupine model — T1 and T2

  The fence is the load-bearing safety property: at no instant may two workers
  hold a token that `ack.lua` will accept for one subscription. T1 checks this with
  `porcupine` (anishathalye/porcupine). The model is a pure, I/O-free sequential
  oracle over one subscription's fence register `(gen, wake, phase)`, partitioned
  by subscription so the NP-hard linearizability search stays per-key. `stepArm`
  requires the generation to strictly increase from idle; `stepClaim` coalesces
  (reuse the exact `gen`/`wake`) only from `phase=waking` with a wake set, else
  rotates strictly upward; and `stepAckOrRelease` computes the fence exactly as the
  Lua does.

  ```go title="model_fence.go — the fence step, the load-bearing safety gate"
  func stepAckOrRelease(s fenceState, in fenceInput, out fenceOutput) (bool, fenceState) {
      fenced := in.tokenGen != s.gen || in.reqGen != s.gen ||
                in.reqWake == "" || in.reqWake != s.wake
      if out.status == statusOK && fenced {
          // two holders' tokens were both honored — no valid linearization.
          return false, s
      }
      // ... apply a legal OK forward; BUSY / NOSUB / FENCED are no-ops
  }
  ```

  Two design choices in the model are worth naming, because an adversarial review
  found a false-`Illegal` without them. Time is deliberately *absent*: `BUSY`,
  `NOSUB`, and crucially `FENCED` are unconditional legal no-ops. A `FENCED` grants
  nothing, so it cannot be half of a violation, and modeling it as an unconditional
  no-op absorbs `expire_lease.lua` clearing `wake_id` without rotating `gen` — an
  invisible server event the history cannot see. The other choice is that
  generations start at 1 in the unit tests, to mirror `create_sub.lua`'s `gen=0`
  plus the first `HINCRBY`.

  The pure model is exercised two ways. The live driver runs N real workers against
  a real subscription, with an in-process `gcPause` nemesis that stalls roughly
  every third grant past `lease_ttl_ms` so a peer takes over and the stalled ack
  races in stale; the recorder brackets each outcome — recording `BUSY`/`FENCED`
  verbatim, not as errors — using the host monotonic clock so `[Call, Return]`
  stays sound under clock skew.

  ```text title="scenario_lease.go — the live T1 driver, in shape"
  N workers, until deadline:
    claim  -> record(opClaim)
    if grants % 3 == 0 { sleep(lease_ttl + 250ms) }   # gcPause: stall past the lease
    ack    -> record(opAck)                            # the stalled one races in stale

  result, info := porcupine.CheckOperationsVerbose(leaseModel(), history, 20s)
  switch result {
    Ok:      pass
    Illegal: VisualizePath(info, "linz-counterexample.html")  # two-holder witness
    Unknown: too concurrent to decide  -> accepted T1 limit (gap #1)
  }
  ```

  The 13 model unit tests pin clauses the live driver cannot reliably reach — a
  stale bearer token where `reqGen == current` but `tokenGen` is stale, the
  post-expiry `FENCED` that must linearize — and run with `CheckOperations` (no
  timeout, so a definitive `Ok`/`Illegal`), no cluster, no Redis.

  ```text title="model_fence_test.go — a stale ack accepted is Illegal"
  h := history{
    claim(A) -> CLAIMED(gen=1, wA),
    claim(B) -> CLAIMED(gen=2, wB),     # B rotated the fence to 2
    ack(A, gen=1, wA, done) -> OK,      # A's token is stale; an OK here is two-holder
  }
  checkLinearizable(t, h, /*want*/ false)   # no valid linearization exists
  ```

  ### A takeover history, drawn out

  The single-holder violation T1 exists to catch is the lease takeover that fails
  to rotate the fence. The diagram below contrasts the broken pre-slice behavior
  (B reuses A's generation, both tokens stay valid — a split brain) with the
  current behavior (B rotates, A is fenced).

  ```text title="Expired-lease takeover — the fence must rotate"
  time ─────────────────────────────────────────────────────────────────▶

  worker A   claim ─────[ holds lease, gen=1, wake=wA ]───X stalled past TTL
                                                          │
                                    lease expires ........│
                                                          ▼
  worker B                              claim (takeover) ─[ gen=?, wake=wB ]── ack ─▶ OK
                                                          │
  worker A (resumes) ............................. ack(gen=1, wA) ─▶ ???

   ┌─ BROKEN (pre-slice): B reuses gen=1, wake=wA ───────────────────────────┐
   │   A's ack(gen=1, wA)  matches current  ->  OK   ← two holders, split brain│
   └──────────────────────────────────────────────────────────────────────────┘

   ┌─ FIXED: B rotates to gen=2, wake=wB ────────────────────────────────────┐
   │   A's ack(gen=1, wA)  tokenGen 1 != cur 2  ->  409 FENCED  ← single holder │
   └──────────────────────────────────────────────────────────────────────────┘
  ```

  T2 is simpler by construction. A single backward step in a cursor is a finite
  counterexample, so it needs no linearization search — it is a direct
  non-decreasing checker over the sampled stream. `CheckCursorMonotonic` keeps a
  per-`(sub, path)` high-water mark; a sample below the mark is one violation but
  keeps the existing mark, so a dip-and-recover reports exactly once rather than
  cascading. Its `offsetGreater` mirrors `common.lua` and the webhook a third time
  — the triple-mirror pattern — with `-1`/`""` as the beginning sentinel.

  ```text title="check_cursor.go — a dip-and-recover is one report, not a cascade"
  samples for one (sub, path):  20 ─▶ 5 ─▶ 21
                                │     │     │
  high-water:                   20    20    20→21
  violation:                          ✗ (5 < 20)
                                            (21 ≥ 20: not re-flagged)
  => exactly 1 violation, not 2
  ```

  <Callout icon="alert" label="What Jepsen does not yet cover">
    <strong>The arm-crash is an approximation.</strong> The few-microsecond
    arm/emit window cannot be hit from an external host driver; it is approximated
    with coarse pod-kill plus churn, and the code itself notes a surgical kill
    needs a server-side fault-injection seam this harness lacks (research doc 07
    proposes <code>gofail</code>).
    <strong>There is no real failover substrate.</strong> <code>killRedis</code>
    deletes the one Redis pod and relies on PVC-backed AOF replay; there is no
    replica, sentinel, or cluster failover, so split-brain and replication-lag
    durability are untested. Acceptance gates T3/T5 and liveness L2/L4/L5 are
    unimplemented until proposed sharding lands, and a porcupine
    <code>Unknown</code> on an over-concurrent history is an accepted T1 limit.
  </Callout>

  The benchmark isolates exactly the cost that scaling the sweep is about: the
  Redis round-trip read cost, with no wakes firing. `benchRedis` pings an isolated
  DB (db 13) and `b.Skipf`s if it is down, so `go test ./...` stays green
  serverless; it seeds K idle subscriptions each linked to P *non-existent* streams
  (a tail `HGET` costs the same round trip whether the stream exists), asserts the
  seeded subs are `PhaseIdle` so the pending-work read path is actually exercised,
  and then loops `RunSweep` over a K/P case matrix.

  ```go title="sweep_bench_test.go — the benchmark in shape"
  if !benchRedis(b) { b.Skipf("redis db 13 unreachable") }   // green serverless
  seedIdleSubs(K, P)                  // P dead links each: same RT as live ones
  for _, sub := range subs {
      require(sub.Phase == PhaseIdle)  // else the read path measures nothing
  }
  b.ResetTimer()
  for i := 0; i < b.N; i++ {
      mgr.RunSweep()                   // one tick: GetMany + TailOffsets, no wakes
  }
  // cases: K1000_P1, K1000_P5, K5000_P5, K10000_P1
  ```

  <Complexity
    time="O(K + U) work · ⌈K/512⌉ + ⌈U/512⌉ round trips per tick"
    space="O(K + U) for the swept ids and the tail map"

  `sweepOnce` uses `GetMany` plus `TailOffsets` (pipelined), replacing one round
  trip per subscription and one per link — at K=10,000 that is roughly 40 pipelined
  round trips against about 20,000 sequential ones in the old form. The conformance
  suite is property-based (fast-check) plus example-based vitest, vendored as the
  pinned npm `@durable-streams/server-conformance-tests@0.3.5` and run black-box

<section class="block" id="findings">
  <SectionHead num="07" label="Findings" title="What the suite <em>does and does not</em> prove" />

  The single sentence that captures the suite: **a pure deterministic core behind
  an imperative I/O shell, each tested at the cheapest faithful level, with the
  emergent cross-process properties model-checked under fault injection.** What
  follows is the honest ledger — what that buys, where it stops, and the
  techniques that do the work.

  ### Strengths

  - **Differential memory/Lua-vs-oracle parity.** One §5.2.1 producer table is the
    oracle; the Redis Lua append path is asserted byte-identical to pure-Go
    `ValidateProducer` per row, including the persisted state and write-iff-accepted.[^f-diff]
  - **Pure-core/imperative-shell discipline.** `model_fence.go` and
    `check_cursor.go` are I/O-free oracles unit-tested with hand-built porcupine
    histories (18 jepsen unit tests, no cluster); the `scenario_*.go` files are the
    live drivers.[^f-split]
  - **Porcupine linearizability gate on the lease fence.** A sequential
    single-holder model encodes the claim/ack Lua; an `OK` ack under a stale token
    is the two-holder violation, surfaced with `VisualizePath` counterexamples.[^f-porcupine]
  - **Bidirectional Lua/Go lockstep mirrors.** `common.lua` `fenced()` and
    `state.go` `FenceDecision` each cite the other; the cursor checker re-mirrors
    `offsetGreater` a third time, so the same rule is pinned from three sides.[^f-mirror]
  - **Property-style lex invariant over adversarial bytes.** Frame/offset tests
    sort `0x00`/`0xff`/`|` payloads and assert bytewise order equals stream order —
    the foundation of the ZSET log model.[^f-lex]
  - **Real-Redis-or-skip with isolated DBs.** Each tier pings a dedicated DB (15
    store, 14 webhook, 13 bench) and skips under `-short` or when unreachable,
    keeping `go test ./...` green serverless while still exercising real Lua and
    round-trip cost.[^f-redis]
  - **Determinism via injected seams.** `TailFunc`, `IPResolver`, explicit `now`
    clocks, and an explicit jitter fraction make fence/claim/retry/SSRF
    reproducible; `fakeStreams`/`fakeLister`/`fakeMetrics` isolate Manager
    orchestration over a real store.[^f-seams]
  - **Conformance plus timing-bound concurrency assertions.** ~330 black-box
    conformance tests pin the wire spec; the store notify tests assert an elapsed
    time under one second proves pub/sub and under 200ms proves the fast path — not
    just the outcome, the mechanism.[^f-timing]

  ### Gaps

  <Table wide>
    | Gap | Severity | Why it matters |
    | --- | --- | --- |
    | Jepsen arm-crash is an approximation | **high** | the few-microsecond arm/emit window can't be hit from an external host; coarse pod-kill + churn stands in, and the code says a surgical kill needs a server-side seam this harness lacks (07 proposes `gofail`)[^g-arm] |
    | Single-node Redis, no real failover | **high** | `killRedis` deletes the one pod and relies on AOF replay; no replica/sentinel/cluster substrate, so split-brain and replication-lag durability are untested[^g-failover] |
    | Lua/Go parity asserted by convention | medium | no test `EVAL`s the Lua and runs the Go mirror on identical inputs for byte-equal output; parity rests on mirrored assertions plus doc comments[^g-parity] |
    | No fuzz/property/linearizability in store or HTTP | medium | despite the repo name, store and handler layers are example-based tables only — no `go test fuzz`, no property harness, no `-race` multi-writer store test[^g-fuzz] |
    | True concurrency simulated sequentially | medium | webhook two-worker races are reproduced by advancing a clock, not running in parallel; only the jepsen single-holder-linz driver runs genuinely concurrent[^g-concurrency] |
    | Acceptance/liveness gates unimplemented | low | jepsen T3/T5 and L2/L4/L5 are red until proposed sharding lands; porcupine `Unknown` on over-concurrent histories is an accepted T1 limit[^g-gates] |
    | Memory-vs-Redis parity is structural | low | both back the same `Store` interface but most contract cases are duplicated per-backend, not run through one shared harness, so divergences rely on hand-kept mirroring[^g-parity-store] |
    | Thin packages; conformance not installed | low | `metrics` has only 2 tests; conformance `node_modules` isn't vendored in-tree, so the ~330 count needs `npm ci` in CI[^g-thin] |
  </Table>

  ### Techniques

  <Table wide>
    | Technique | What it does | Code ref |
    | --- | --- | --- |
    | Differential / oracle parity | the pure-Go `ValidateProducer` table is the single source of truth; the Lua append path is asserted byte-identical row-by-row, incl. persisted state and write-iff-accepted | [^t-diff] |
    | Porcupine linearizability | a pure sequential fence `Model` (partition-by-sub) checks recorded concurrent claim/ack histories for single-holder; `Illegal` emits an HTML counterexample | [^t-porcupine] |
    | Lua/Go pure-mirror lockstep | `common.lua` `fenced()` and `state.go` `FenceDecision` cite each other; `check_cursor` re-mirrors `offsetGreater` a third time — pinned from Go, Lua, and the checker | [^t-mirror] |
    | Fault injection on k3d | a `kubectl`/`redis-cli` nemesis (`killOneOrigin`, `killAllOrigins`, `killRedis`, `deleteStreamIndex`) drives crash scenarios; final-state checks assert every cursor reaches tail within `-settle` | [^t-fault] |
    | In-process gcPause nemesis | stalls a claimed token past `lease_ttl_ms` so a peer takes over and the stale ack races in; highest-ROI T1 fault, needs no infra, host monotonic clock keeps `[Call, Return]` sound | [^t-gcpause] |
    | Property-style lex invariant | frame/offset tests sort adversarial `0x00`/`0xff`/`|` payloads and assert bytewise order equals stream offset order — the ZSET log model's foundation | [^t-lex] |
    | Real-Redis-or-skip, isolated DBs | store/webhook/bench each ping a dedicated DB (15/14/13), `FlushDB` once, and skip under `-short` or when unreachable; no miniredis anywhere | [^t-redis] |
    | Golden-substring + closed-form invariants | loadgen renders Markdown/K8s/TF artifacts checked by substring; pace tests invert the linear-ramp cumulative-hits integral; HDR quantiles checked to ~0.1% | [^t-golden] |
  </Table>

  The through-line is that the most safety-critical rules — the producer fence,
  the subscription fence, the forward-only cursor, the lex order under the log —
  are each pinned from more than one independent angle. A regression that wants to
  ship has to defeat the pure oracle, its mirror, and where it matters a recorded
  concurrent history all at once. What it does *not* yet have is a real failover
  substrate or a surgical crash seam, and it leans on convention where it could
  enforce equivalence. Those are the named next steps, not silent omissions.
</section>
